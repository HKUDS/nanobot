import json
import re

class SubmissionEvaluator:
    # ... 其他方法保持不变 ...

    async def _call_container_summarize_skill(self, container_url: str, bounty_desc: str, submission_content: str) -> dict:
        url = f"{container_url}/summarize_skill"
        payload = {
            "bounty_description": bounty_desc,
            "submission_content": submission_content
        }
        headers = {"Content-Type": "application/json"}

        async with aiohttp.ClientSession(timeout=aiohttp.ClientTimeout(total=90)) as session:
            async with session.post(url, json=payload, headers=headers) as resp:
                if resp.status != 200:
                    error_text = await resp.text()
                    raise Exception(f"容器 Skill 总结返回错误: {resp.status}, {error_text}")

                raw_text = await resp.text()
                print(f"[Evaluator] 容器返回原始文本 (前200字符): {raw_text[:200]}")

                # 1. 预处理：去除 BOM 头和首尾空白
                cleaned_text = raw_text.strip().lstrip('\ufeff')

                # 2. 尝试提取 JSON (兼容代码块包裹)
                json_str = cleaned_text
                if cleaned_text.startswith('```'):
                    # 移除 markdown 代码块标记
                    json_str = re.sub(r'^```(?:json)?\s*', '', cleaned_text)
                    json_str = re.sub(r'\s*```$', '', json_str)

                # 3. 安全解析 JSON
                try:
                    data = json.loads(json_str)
                except json.JSONDecodeError as e:
                    print(f"[Evaluator] JSON 解析失败: {e}")
                    # 降级处理：尝试使用正则提取 name 字段
                    name_match = re.search(r'"name"\s*:\s*"([^"]+)"', cleaned_text)
                    fallback_name = name_match.group(1) if name_match else "skill-fallback"
                    print(f"[Evaluator] 降级提取技能名: {fallback_name}")
                    return {
                        "name": fallback_name,
                        "description": "解析响应失败，已使用降级策略",
                        "usage": "",
                        "instructions": cleaned_text[:500]
                    }

                # 4. 兼容两种响应格式
                # 格式一：{"skill_summary": {...}}
                # 格式二：直接返回 skill 对象 {...}
                if "skill_summary" in data:
                    skill_data = data["skill_summary"]
                else:
                    skill_data = data

                # 确保必需字段存在
                skill_summary = {
                    "name": skill_data.get("name", "unknown"),
                    "description": skill_data.get("description", ""),
                    "usage": skill_data.get("usage", ""),
                    "instructions": skill_data.get("instructions", ""),
                    "examples": skill_data.get("examples", ""),
                    "code_template": skill_data.get("code_template", "")
                }

                print(f"[Evaluator] 解析成功，技能名称: {skill_summary['name']}")
                return skill_summary
```

### 🔴 问题 1：JSON 解析失败 (BOM 头问题)

**日志证据**:
```
[_generate_skill_summary] JSON 解析失败: Expecting property name enclosed in double quotes: line 1 column 2 (char 1), 原始内容: {
    "name": "tech-news-research-reporting",
    ...
```
错误信息 `Expecting property name enclosed in double quotes` 且指向 `line 1 column 2 (char 1)`，这表明第一个字符不是 `{`，而是一个不可见的控制字符，最有可能是 **UTF-8 BOM (Byte Order Mark)** (`\ufeff`)。

**代码位置**: `evaluator.py` 或容器内处理 `/summarize_skill` 的逻辑。

**原因分析**:
1.  LLM 返回的文本流可能被容器错误地编码，或者响应头未正确处理，导致 BOM 被保留。
2.  直接对包含 BOM 的字符串调用 `json.loads()` 会引发上述异常。

**修复方案**:
在解析 JSON 之前，必须清除 BOM 和首尾空白。
```python
def _safe_json_loads(content: str):
    # 移除 BOM 和首尾空白
    cleaned = content.strip().lstrip('\ufeff')
    return json.loads(cleaned)
```

### 🟡 问题 2：Skill 总结接口响应结构不匹配

**日志证据**:
```
[EvaluateBatch] ✅ 提交 ... 的Skill总结成功，技能名称: unknown
```
即使 JSON 解析成功，提取到的名称却是 `"unknown"`。这说明 `evaluator.py` 中的 `_call_container_summarize_skill` 方法未能从响应中正确提取 `name` 字段。

**代码位置**: `evaluator.py` -> `_call_container_summarize_skill`

**当前逻辑推测**:
```python
data = await resp.json()
skill_summary = data.get("skill_summary", {})
```
期望容器返回格式: `{"skill_summary": {"name": "...", ...}}`

**实际情况 (由日志推断)**:
容器直接返回了 JSON 对象本身，而不是包装在 `skill_summary` 字段中。
```json
{
    "name": "tech-news-research-reporting",
    "description": "...",
    ...
}
```
因此 `data.get("skill_summary")` 得到 `None` 或空字典，最终名称变成默认值 `"unknown"`。

**修复方案**:
修改 `evaluator.py` 的解析逻辑，使其兼容两种格式：
```python
data = await resp.json()
# 兼容直接返回对象的情况
if "skill_summary" in data:
    skill_summary = data["skill_summary"]
else:
    skill_summary = data  # 假设整个响应就是 skill_summary
```
