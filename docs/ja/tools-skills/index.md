# ツールとスキル概要

Nanobot の拡張は 2 つのレイヤに分かれます。**ツール（Tools）** はエージェントが実行できる内蔵の操作能力で、**スキル（Skills）** は配布可能な知識・手順（ワークフロー）モジュールです。両者が役割分担し、柔軟で拡張可能な AI アシスタントを構成します。

---

## ツール（Tools）

ツールはエージェントが直接呼び出す低レベル能力で、具体的な操作に対応します（Shell 実行、ファイルの読み書き、ウェブ検索、スケジューリングなど）。ツールは nanobot コアが提供し、設定で有効化・挙動調整できます。

| ツール名 | 説明 |
|---------|------|
| `exec` | Shell コマンドとスクリプトを実行 |
| `read_file` | ファイル内容を読み取り（ページング対応） |
| `write_file` | ファイルへ書き込み（ディレクトリ自動作成） |
| `edit_file` | ファイル中のテキスト断片を正確に置換 |
| `list_dir` | ディレクトリ構造を列挙 |
| `web_search` | ウェブ検索（複数プロバイダ） |
| `web_fetch` | ウェブページ内容を取得して解析 |
| `cron` | リマインダー/定期タスクのスケジューリング |
| `spawn` | バックグラウンドでサブエージェントを起動して複雑タスクを実行 |
| `message` | ユーザーへメッセージ送信（添付対応） |

詳細は [ツール利用ガイド](tools.md) を参照してください。

---

## スキル（Skills）

スキルは Markdown でパッケージ化された知識とワークフローで、エージェントに「どうやってやるか」を教えます。トリガーされるとエージェントのコンテキストへロードされ、特定領域の手順、コマンド例、再利用可能なスクリプトや参考資料を提供します。

スキルは [OpenClaw](https://github.com/openclaw/openclaw) の仕様に準拠しており、OpenClaw のスキルエコシステムと互換です。

| スキル名 | 説明 |
|---------|------|
| `github` | `gh` CLI で GitHub と連携 |
| `weather` | 天気情報を取得（wttr.in / Open-Meteo） |
| `summarize` | URL、YouTube、ローカルファイルを要約 |
| `tmux` | tmux セッションを遠隔操作 |
| `memory` | 2 層メモリ（長期事実 + 履歴） |
| `cron` | リマインダー/定期タスクの運用ガイド |
| `clawhub` | ClawHub の公開スキルを検索・インストール |
| `skill-creator` | カスタムスキル作成の完全ガイド |

詳細は [スキル利用ガイド](skills.md) を参照してください。

---

## ツールとスキルの違い

| 観点 | ツール（Tools） | スキル（Skills） |
|------|--------------|---------------|
| 本質 | Python コード。エージェントが直接呼び出せる | Markdown。エージェントのコンテキストにロードされる |
| 拡張方法 | nanobot コアを変更するか MCP を利用 | 誰でも `.md` を書ける |
| トリガー | エージェントが自律的に選択して呼び出す | `description` フィールドをトリガーにロード |
| 提供するもの | 操作を実行（Shell/ウェブ/ファイル…） | 知識、手順、例 |
| 配布 | nanobot と一緒にインストール | `.skill` パッケージとして配布可能（ClawHub に投稿可能） |

---

## MCP 統合

MCP（Model Context Protocol）は外部ツールサーバーに接続するためのオープンプロトコルです。MCP を使うと、nanobot はコアコードを変更せずに任意の外部ツールを動的にロードできます。

MCP サーバーは次の 2 方式で接続できます。

- **Stdio**: ローカルプロセス。標準入出力で通信
- **HTTP/SSE**: リモートサービス。URL で接続

各 MCP ツールは nanobot 内で `mcp_<サーバー名>_<ツール名>` の形式で現れ、内蔵ツールと同じようにエージェントが利用します。

詳細は [MCP 統合ガイド](mcp.md) を参照してください。
