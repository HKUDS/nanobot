## 飞书机器人配置步骤

### 1. 创建飞书机器人应用

1. 访问 [飞书开放平台](https://open.feishu.cn/app)
2. 点击"创建企业自建应用"
3. 填写应用名称和描述
4. 创建完成后，进入应用管理页面

### 2. 配置机器人权限

在应用管理页面：

1. **启用机器人功能**
   - 进入"添加应用能力" → 启用"机器人"

2. **添加权限**
   - 进入"权限管理"
   - 添加以下权限：
     - `im:message` - 发送消息
     - `im:message.p2p_msg:readonly` - 接收私聊消息
     - `im:message.group_msg:readonly` - 接收群聊消息（如需要）

3. **配置事件订阅**
   - 进入"事件订阅"
   - 选择"长连接模式"（WebSocket）
   - 添加事件：`im.message.receive_v1` - 接收消息

### 3. 获取应用凭证

在"凭证与基础信息"页面获取：
- **App ID** (格式：`cli_xxx`)
- **App Secret**

可选（长连接模式不需要）：
- Encrypt Key（加密密钥）
- Verification Token（验证令牌）

### 4. 发布应用

1. 在应用管理页面点击"版本管理与发布"
2. 创建版本并提交审核
3. 审核通过后发布到企业

### 5. 配置 nanobot

编辑配置文件 `~/.nanobot/config.json`，找到 `feishu` 部分：

```json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "appId": "cli_你的AppID",
      "appSecret": "你的AppSecret",
      "encryptKey": "",
      "verificationToken": "",
      "allowFrom": ["ou_你的OpenID"],
      "reactEmoji": "THUMBSUP"
    }
  }
}
```

**配置说明：**
- `enabled`: 设置为 `true` 启用飞书频道
- `appId`: 填入你的 App ID
- `appSecret`: 填入你的 App Secret
- `encryptKey`: 长连接模式可留空
- `verificationToken`: 长连接模式可留空
- `allowFrom`: 允许交互的用户 Open ID 列表
  - 使用 `["*"]` 允许所有用户
  - 或填入具体的 Open ID（首次运行时可在日志中看到）
- `reactEmoji`: 收到消息时的表情反应（可选：THUMBSUP, OK, EYES, DONE, OnIt, HEART）

### 8. 测试连接

1. 在飞书中搜索你的机器人名称
2. 发送一条消息给机器人
3. 查看 nanobot 日志，应该能看到：
   - WebSocket 连接成功

### 获取你的 Open ID

首次运行时：
1. 启动 `nanobot gateway`
2. 在飞书中向机器人发送消息
3. 查看日志，会显示发送者的 Open ID
4. 将该 Open ID 添加到 `allowFrom` 列表中
5. 重启网关

## 常见问题

### Q: 如何让机器人在群聊中工作？
A: 需要添加 `im:message.group_msg:readonly` 权限，并将机器人添加到群聊中。

### Q: 长连接模式的优势是什么？
A: 不需要公网 IP 或 webhook，使用 WebSocket 接收消息，配置更简单。

### Q: 如何查看日志？
A: 运行 `nanobot gateway` 时会在终端显示实时日志。

### Q: 支持哪些消息类型？
A: 支持文本、图片、文件、音频、富文本等多种消息类型。
