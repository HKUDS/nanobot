## 一、实验目标

验证 **SAyG‑Mem（分段并发写入 + 异步合并）** 相对于 **传统并发加锁写入 + 轮间同步合并方案**，在相同多 Agent 学习任务下的**时间效率**与**知识质量**优势。

---

## 二、实验组设计

| 组别 | Agent 推理 | 写入方式 | 合并机制 | 轮间约束 |
| :--- | :--- | :--- | :--- | :--- |
| **A 组（SAYG-Mem）** | 5 个并发 | 各自写入独立 Heap 文件，无锁追加 | 后台 Consolidator 异步合并 + SimHash 去重 | 无：Agent 完成一轮后立即进入下一轮 |
| **B 组（并发推理 + 写入加锁 + 轮间同步合并）** | 5 个并发 | 共同写入同一个 PublicMemory 文件，使用 `fcntl.flock` 排队 | **每轮结束后**，主控线程调用 LLM 对本轮所有 Agent 产出进行合并，并将合并结果写入 PublicMemory（原产出也保留） | **必须等待本轮全部 Agent 完成写入、且 LLM 合并完成**，才启动下一轮 |

> **核心对比**：两组均为并发推理，唯一变量是**写入机制**与**合并时机**。A 组写入无锁、合并异步；B 组写入加锁排队、合并同步阻塞。

### 4.2 A 组（SAYG-Mem）执行流程

```python
# A组伪代码
async def run_group_a(agents, rounds, prompts):
    # 1. 启动 Consolidator 后台线程
    consolidator = Consolidator(heap_dir=HEAP_DIR, public_memory_path=PM_PATH)
    consolidator.start()

    # 2. 并发启动所有 Agent
    start_time = time.perf_counter()
    tasks = [agent_run_5_rounds(agent, prompts) for agent in agents]
    await asyncio.gather(*tasks)

    # 3. 等待 Consolidator 处理完队列
    while consolidator.queue_size() > 0:
        await asyncio.sleep(1)
    consolidator.stop()

    end_time = time.perf_counter()
    return end_time - start_time
```

### 4.3 B 组（并发推理 + 写入加锁 + 轮间同步合并）执行流程

**关键机制**：
1. Agent 推理并发执行。
2. 写入 PublicMemory 时使用文件锁（`fcntl.flock`）排队。
3. 每轮所有 Agent 完成后，主控线程调用 LLM 对本轮产出进行合并，合并结果也通过加锁写入 PublicMemory。
4. 合并完成后，才启动下一轮。

```python
# B组伪代码
import fcntl

def write_with_lock(content: str):
    """带文件锁的写入"""
    lock_path = "./data/public_memory/public_memory.lock"
    with open(lock_path, 'w') as lf:
        fcntl.flock(lf, fcntl.LOCK_EX)  # 获取排他锁
        with open(PM_PATH, 'a') as f:
            f.write(json.dumps({"content": content}) + '\n')
        fcntl.flock(lf, fcntl.LOCK_UN)  # 释放锁

async def run_group_b(agents, rounds, prompts):
    start_time = time.perf_counter()
    for round_idx in range(rounds):
        print(f"B组 第{round_idx+1}轮开始")
        round_start = time.perf_counter()

        # 1. 并发推理，但写入时加锁排队
        tasks = []
        for agent in agents:
            task = asyncio.create_task(agent_round_with_lock(agent, prompts[round_idx]))
            tasks.append(task)
        round_results = await asyncio.gather(*tasks)  # 等待本轮所有 Agent 完成

        # 2. 本轮所有 Agent 写入完成后，调用 LLM 进行合并
        merged_content = await call_llm_merge(round_results)
        # 合并结果也通过加锁写入
        write_with_lock(f"[MERGE] Round {round_idx+1}: {merged_content}")

        round_end = time.perf_counter()
        print(f"B组 第{round_idx+1}轮完成，耗时 {round_end - round_start:.2f}s")

    end_time = time.perf_counter()
    return end_time - start_time
```

**Agent 单轮逻辑（带锁写入）**：
```python
async def agent_round_with_lock(agent_id, role, prompt):
    # LLM 推理（并发）
    resp = await call_llm(agent_id, prompt, role)

    # 写入 PublicMemory（加锁排队）
    write_with_lock(json.dumps({
        "agent_id": agent_id,
        "role": role,
        "round": current_round,
        "page_content": resp["page_content"],
        "page_title": resp["page_title"]
    }))

    return resp
```

**LLM 合并函数**（模拟传统汇总）：
```python
async def call_llm_merge(results: list) -> str:
    prompt = f"请对以下{len(results)}个Agent的产出进行总结合并，保留关键观点，去除重复：\n"
    for r in results:
        prompt += f"- {r['page_content']}\n"
    # 调用 LLM
    merged = await call_llm("system", prompt, role="merger")
    return merged
```

## 五、评价指标与测量方法

| **写入延迟** | P99 写入耗时 | 记录每次 `write_with_lock` 前后时间戳 | <10ms | 50-150ms（锁等待） |

## 八、结论

本方案通过 **A 组（SAyG‑Mem 分段无锁 + 异步合并）** 与 **B 组（传统并发推理 + 写入加锁 + 轮间同步合并）** 的对比，能够精准量化 SAyG‑Mem 在**写入延迟**（消除锁竞争）和**知识质量**（智能去重）两方面的优势。实验设计隔离了推理并发度的影响，结论更具科学性和说服力。
