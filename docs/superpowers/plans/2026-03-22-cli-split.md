# CLI Split Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Split `nanobot/cli/commands.py` (2582 lines) into 8 focused sub-modules with a shared `_make_agent_loop()` factory, leaving `commands.py` as a ~80-line assembly file.

**Architecture:** Each Typer sub-app becomes its own module. Shared factories and utilities move to `_shared.py`. `commands.py` becomes an assembly file that creates the root `app`, imports sub-modules, and registers them. All existing test `app` imports continue working.

**Tech Stack:** Python 3.10+, Typer, pytest, ruff, mypy

```python
def _make_agent_loop(
    config: Config,
    *,
    bus: MessageBus | None = None,
    cron_service: CronService | None = None,
    session_manager: SessionManager | None = None,
) -> AgentLoop:
    """Construct an AgentLoop with standard wiring."""
    from nanobot.agent.loop import AgentLoop
    from nanobot.bus.queue import MessageBus

    provider = _make_provider(config)
    if bus is None:
        bus = MessageBus()
    return AgentLoop(
        bus=bus,
        provider=provider,
        config=_make_agent_config(config),
        brave_api_key=config.tools.web.search.api_key or None,
        exec_config=config.tools.exec,
        cron_service=cron_service,
        session_manager=session_manager,
        mcp_servers=config.tools.mcp_servers,
        channels_config=config.channels,
        routing_config=config.agents.routing,
    )
```

IMPORTANT: Read the actual `AgentLoop.__init__` signature in `nanobot/agent/loop.py` to verify parameter names. The spec provides approximate names — verify against the real constructor.

Use deferred imports (inside function bodies) for heavy internal modules — this is the existing pattern in `commands.py` to keep CLI startup fast.
