# 策略设计方法论

## 概述

策略设计是量化投资的核心环节。一个成功的策略需要清晰的逻辑、严谨的验证和持续的迭代。本技能涵盖从构思到验证的完整方法论。

## 策略构思来源

### 学术研究驱动
- 阅读量化金融学术论文
- 复现论文中的因子或策略
- 基于理论改进现有方法

### 市场观察驱动
- 发现市场的异常现象或规律
- 验证民间投资经验（量化验证）
- 跟随机构持仓或资金流向

### 数据驱动
- 数据挖掘寻找模式（需严谨验证）
- 机器学习发现规律（需过拟合防范）
- 跨市场/跨资产迁移

## 逻辑驱动 vs 数据驱动

### 逻辑驱动策略
- **特点**：先有假设，再验证
- **优势**：逻辑清晰，可解释性强
- **代表**：基于财务因子的价值策略
- **风险**：假设可能不成立

### 数据驱动策略
- **特点**：从数据中发现规律
- **优势**：可能发现人类难以察觉的模式
- **代表**：机器学习预测模型
- **风险**：过拟合风险高

### 建议
- 优先逻辑驱动，数据驱动作为补充
- 任何策略都需要有经济逻辑支撑

## 多空辩论提示词

在设计新策略时，使用以下框架进行自问自答：

```
## 策略核心逻辑
这个策略的核心假设是什么？

## 支撑证据
什么数据或理论支撑这个假设？

## 反对观点
为什么这个策略可能不work？

## 失效情景
什么情况下策略会失效？

## 竞争对比
与现有策略相比优势在哪里？
```

### 辩论检查清单

| 问题 | 是 | 否 |
|------|----|----|
| 逻辑是否可解释？ | 继续 | 重新设计 |
| 是否有经济含义？ | 继续 | 谨慎 |
| 样本外验证通过？ | 继续 | 调整或放弃 |
| 成本考虑充分？ | 继续 | 重新计算 |
| 容量是否足够？ | 继续 | 调整规模 |

## 策略迭代流程

### 第一阶段：概念验证
- 用最简模型测试核心逻辑
- 使用月度数据快速验证
- 目标：确认方向正确

### 第二阶段：参数优化
- 确定关键参数和搜索范围
- 使用样本内数据优化
- 样本外数据验证

### 第三阶段：稳健性测试
- 参数敏感性分析
- 蒙特卡洛模拟
- 极端行情测试

### 第四阶段：成本优化
- 计算真实成本影响
- 优化交易执行
- 考虑流动性约束

### 第五阶段：实盘准备
- 模拟盘测试
- 监控系统搭建
- 风险管理规则

## 常见策略类型

### 趋势跟踪策略
- **逻辑**：趋势会延续
- **入场**：价格突破N日高点
- **出场**：价格跌破N日低点
- **适用**：趋势明显的市场
- **风险**：震荡市场频繁止损

### 均值回归策略
- **逻辑**：价格会回归价值
- **入场**：价格偏离均值达到阈值
- **出场**：价格回归均值
- **适用**：波动较大的市场
- **风险**：趋势形成后可能持续偏离

### 套利策略
- **跨品种套利**：相关品种价差交易
- **跨期套利**：期货近远月价差
- **跨市场套利**：不同市场间价差
- **特点**：低风险、低收益、高频
- **风险**：流动性、时滞

### 多空策略
- **Long-Short**：做多低估，做空高估
- **Market Neutral**：对冲市场风险
- **130/30**：130%多头+30%空头
- **特点**：降低市场风险暴露

### 量化因子策略
- **多因子模型**：因子加权组合
- **风险平价**：风险分配权重
- **因子轮动**：动态因子配置
- **特点**：系统性、可复制

## 策略评估框架

### 收益维度
- 绝对收益
- 相对收益（vs基准）
- 风险调整收益

### 风险维度
- 最大回撤
- 波动率
- 尾部风险

### 能力维度
- 信息比率
- 选股能力
- 时机把握能力

### 成本维度
- 交易成本占比
- 换手率
- 容量

## 策略设计常见错误

1. **过度复杂**：用过多参数拟合历史
2. **忽视成本**：低成本假设导致虚高收益
3. **不回测成本**：只在逻辑层面考虑
4. **不验证逻辑**：直接假设策略有效
5. **忽视流动性**：大资金无法按回测价成交
6. **幸存者偏差**：只用当前股票回测

## 建议实践

- 先有清晰逻辑，再写代码回测
- 保持策略简洁，参数越少越好
- 始终考虑最坏情况
- 收益预期合理（年化15-30%已属优秀）
- 持续记录和复盘
