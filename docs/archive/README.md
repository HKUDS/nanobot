# Archive

Historical documents from the nanobot project. These are point-in-time records
retained for context -- they describe past states of the codebase and should NOT
be updated to match current code.

## Directory Structure

- `reviews/` — Architecture reviews, security audits, CI/CD reviews
- `proposals/` — Design proposals that were implemented or superseded

Superpowers specs, plans, and reports stay in `docs/superpowers/` as part of
the active development workflow. They are NOT archived here.
