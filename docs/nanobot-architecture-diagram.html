nanobot Architecture Diagram

Unified agent runtime across CLI, API, WebSocket/WebUI, and multi-channel connectors

Entry Modes

  • • CLI (`nanobot agent`) and Python SDK
  • • OpenAI-compatible HTTP API (`serve`)
  • • WebSocket/WebUI and multi-chat channels (`gateway`)

Core Runtime

  • • `AgentLoop` orchestrates sessions and dispatch
  • • `AgentRunner` executes model/tool iterative loop
  • • Tool layer supports filesystem, exec, web, MCP, spawn

State & Automation

  • • Session history in JSONL with checkpoint recovery
  • • Memory + Dream consolidation pipeline
  • • Cron and heartbeat drive long-running agent behavior
