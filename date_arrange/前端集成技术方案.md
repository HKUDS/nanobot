# Date Arrange 前端集成技术方案

## 📋 项目概述

基于 Docker 容器化的智能日程规划系统，将 Date Arrange 模块与 Nanobot 系统深度集成，提供完整的自然语言日程规划解决方案。

## 🏗️ 系统架构

### 整体架构图

```
用户界面 (Vue 3 + Element Plus)
        ↓
Nginx 反向代理 (负载均衡)
        ↓
┌─────────────────┬─────────────────┐
│  前端服务       │  后端API服务    │
│ (Vite + Vue)    │ (FastAPI)       │
└─────────────────┴─────────────────┘
        ↓
┌─────────────────────────────────────┐
│          Nanobot Agent 服务         │
│        (Docker 容器化)              │
└─────────────────────────────────────┘
```

### 技术栈

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Vue 3 + Element Plus + Vite | 现代化前端框架，响应式设计 |
| 后端 | FastAPI + Pydantic | 高性能异步API框架 |
| 容器化 | Docker + Docker Compose | 微服务架构部署 |
| 代理 | Nginx | 反向代理和负载均衡 |
| AI集成 | Nanobot Agent | 智能对话和记忆管理 |

## 🎯 核心功能

### 1. 智能任务解析
- **自然语言输入**: 用户通过自然语言描述日程需求
- **智能解析**: 自动识别任务、优先级、预计时长
- **上下文理解**: 结合对话上下文进行任务分析

### 2. 优化日程安排
- **智能调度**: 基于优先级和约束的日程优化
- **时间段分配**: 自动分配任务到合适的时间段
- **效率评估**: 提供日程效率评分和改进建议

### 3. Nanobot集成
- **记忆管理**: 将日程信息存储到Nanobot记忆系统
- **智能对话**: 通过Nanobot进行自然语言交互
- **持续学习**: 基于用户反馈优化日程规划

### 4. 可视化界面
- **任务展示**: 卡片式任务列表，优先级颜色标识
- **日程视图**: 时间轴式日程安排展示
- **实时交互**: 拖拽调整、实时预览功能

## 🚀 部署方案

### 开发环境部署

```bash
# 1. 克隆项目
git clone <repository>
cd date_arrange

# 2. 配置环境变量
cp .env.example .env
# 编辑.env文件，配置API密钥等参数

# 3. 启动开发环境
# 后端服务
cd api && uvicorn main:app --reload --port 8000

# 前端服务  
cd frontend && npm install && npm run dev
```

### 生产环境部署

```bash
# 使用Docker Compose一键部署
./deploy.sh  # Linux/macOS
deploy.bat   # Windows

# 或手动部署
docker-compose up -d
```

### 服务配置

| 服务 | 端口 | 说明 |
|------|------|------|
| Nginx | 80 | 反向代理，统一入口 |

## 🔧 核心组件设计

### 1. 前端架构

```
src/
├── views/           # 页面组件
│   ├── SchedulePlanner.vue    # 日程规划主界面
│   └── TaskManager.vue        # 任务管理界面

│   ├── TimeSlot.vue           # 时间段组件
│   └── ScheduleView.vue       # 日程视图组件
├── stores/          # 状态管理
│   └── schedule.js            # 日程状态管理
├── api/             # API接口
│   └── schedule.js            # 日程API封装
└── router/          # 路由配置
    └── index.js               # 路由定义
```

### 2. 后端API设计

#### 核心接口

```python
# 任务解析接口
POST /api/schedule/parse
{
    "user_input": "需要准备下周的学术报告",
    "conversation_id": "可选，用于Nanobot集成",
    "constraints": {"available_time": 480}
}

# 日程创建接口
POST /api/schedule/create
{
    "tasks": [...],
    "date": "2024-04-08",
    "constraints": {...}
}

# Nanobot集成接口
POST /api/nanobot/schedule
{
    "conversation_id": "对话ID",
    "user_message": "用户消息",
    "context": {...}
}
```

### 3. 数据流设计

```
用户输入 → 前端收集 → API解析 → 任务处理 → 日程优化 → 结果返回
    ↓
Nanobot记忆存储 ← 结果展示 ← 前端渲染 ← API返回
```

- **代码review** (普通, 60分钟)
- **健身** (低优先级, 30分钟)

✅ 成功解析出3个任务
📅 已创建优化日程
💡 建议：时间利用率较高，效率评分0.82

Nanobot: 已为您创建优化日程，建议将高优先级任务安排在上午...
