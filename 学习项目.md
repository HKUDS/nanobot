User: 我要一级一级学习这个项目。bridge是？case？nanobot等一目录作用

GitHub Copilot: 可以按“从外到内”一级一级看，这个项目顶层目录大致是这几层：

**第0层：先认顶层目录（你问的 bridge/case/nanobot）**
1. nanobot  
主程序代码目录，核心都在这里（CLI、Agent、Channel、Provider、Tools、Session、Config）。
2. bridge  
给某些外部平台做“桥接服务”的 Node/TypeScript 子项目（典型是 WhatsApp）。  
Python 主程序通过它间接接入平台，不是核心推理逻辑。
3. case  
演示素材/示例资源（README 里 GIF、效果展示相关）。偏展示，不是运行主逻辑。
4. tests  
pytest 测试，验证各模块行为。学习时可反向看“这个模块应该怎么工作”。

**第1层：再看 nanobot 内部模块分工**
1. cli  
命令入口，`nanobot agent`、`nanobot gateway` 都从这里发起。
2. agent  
核心大脑：组 prompt、调用模型、执行 tool、循环控制。
3. channels  
各聊天平台适配层（qq/telegram/slack...），负责收发消息。
4. providers  
模型提供方适配（openai/deepseek/litellm 等）。
5. tools  
本地工具能力（读写文件、exec、web、message、cron...）。
6. session、memory.py  
会话与记忆持久化。
7. config  
配置加载、路径解析、schema 定义。

**第2层：你当前最该先读的“最小主链路”**
1. __main__.py  
2. commands.py  
3. loop.py  
4. context.py  
5. base.py + qq.py（你用 QQ）  
6. `nanobot/providers/*`（你当前 provider 对应那个实现）

这条链路看懂后，项目 70% 就通了。

**第3层：bridge 和 case 到底怎么定位**
1. bridge：  
“外围连接器”。你可以把它理解成“某些平台 SDK/协议不方便直接用 Python 时，用 TS 做适配层”。
2. case：  
“展示层资源”。学习代码优先级最低，除非你要补文档/演示。

**推荐学习顺序（一级一级）**
1. 先只看 cli + agent（跑通单机 `agent`）  
2. 再看 qq.py（对接你正在用的 QQ）  
3. 再看 providers（模型调用细节）  
4. 再看 tools 和 skills（扩展能力）  
5. 最后看 bridge 和 tests（工程化与边缘适配）
