# 悬赏市场前端显示问题分析与修复计划

## 问题描述
用户反馈"悬赏发布之前前端没法正常显示"，具体表现为 BountyMarket 组件无法正确加载或显示。

## 根因分析

### 1. 前端组件导入问题
**问题**：BountyMarket 组件使用了错误的 API 调用方式
- **当前代码**：`import request from '../api/agentBff.js'`
- **问题**：直接使用 `request.get('/bounties')` 而不是使用已定义的 `listBounties()` 函数
- **影响**：可能导致 API 调用失败或响应处理错误

### 2. API 响应格式问题
**问题**：BFF 服务返回的格式与前端期望可能不匹配
- **BFF 响应**：`{"bounties": [...]}`
- **前端处理**：`bounties.value = res.data.bounties || []`
- **潜在问题**：如果 `res.data` 结构不符合预期，会导致 `bounties` 为空数组

### 3. 数据库初始化问题
**问题**：bounties 表可能未正确初始化或缺少数据
- **表结构**：包含 `status` 字段，默认为 'open'
- **查询条件**：`status = 'open' AND deadline > ?`
- **影响**：如果表为空，返回空列表，前端显示"暂无悬赏任务"

## 修复计划

### 1. 修复前端 API 调用
**文件**：`frontend/src/components/BountyMarket.vue`
**修改**：
- 导入 `listBounties` 函数
- 使用 `listBounties()` 替代直接的 `request.get('/bounties')`
- 同样修改其他 API 调用（创建悬赏、提交方案等）

### 2. 增强错误处理和日志
**文件**：`frontend/src/components/BountyMarket.vue`
**修改**：
- 在 `fetchBounties` 中添加详细的错误日志
- 显示用户友好的错误提示
- 处理 API 响应格式异常的情况

### 3. 检查 BFF 服务响应
**文件**：`bff/bff_service.py`
**修改**：
- 确保 `api_list_bounties` 函数返回正确的格式
- 添加调试日志，记录返回的 bounty 数量
- 确保数据库连接正常

### 4. 测试数据初始化
**操作**：
- 在数据库中插入测试数据，确保前端能显示至少一个悬赏任务
- 验证 API 响应格式正确

## 执行步骤

1. **修改 BountyMarket.vue**
   - 导入正确的 API 函数
   - 修改 API 调用方式
   - 添加错误处理和日志

2. **检查 BFF 服务**
   - 确保 API 端点返回正确格式
   - 添加调试日志

3. **测试数据初始化**
   - 插入测试悬赏数据
   - 验证前端显示

4. **验证修复效果**
   - 启动服务，测试前端显示
   - 验证悬赏列表加载正常
   - 测试发布和提交功能

## 预期结果
- BountyMarket 组件正常显示
- 即使没有悬赏任务，也能显示"暂无悬赏任务"的提示
- API 调用错误时显示友好的错误提示
- 悬赏列表加载和刷新正常

## 风险评估
- **低风险**：修改仅涉及前端 API 调用方式和错误处理
- **无破坏性**：不影响现有功能，只是增强稳定性和用户体验
- **可回滚**：如果出现问题，可快速恢复到原始代码
