# 悬赏结算结果前端展示技术方案

## 一、需求概述

目前后端已在 `/bounties/{bounty_id}/close` 接口中返回完整的结算结果，包括：
- 每个提交的评分（`score`）和评分理由（`score_reason`）
- 奖励分配情况（`reward_amount`）
- 评级等级（`final_grade`，S/A/B/C/D）

需要在前端悬赏详情或结算后展示这些信息，让发布者和参与者清晰看到结算明细。

## 二、现有数据基础

### 2.1 后端返回的结算结果结构

```json
{
  "status": "completed",
  "submissions_count": 3,
  "evaluation_results": [
    {
      "submission_id": "xxx",
      "agent_id": "node_a",
      "score": 85.0,
      "reason": "内容完整，可执行性强",
      "reward_amount": 5000,
      "status": "success"
    }
  ],
  "reward_results": [ ... ],
  "curation_results": null
}
```

### 2.2 前端已有组件
- `BountyMarket.vue`：悬赏市场主页面，包含悬赏列表、发布、提交、查看方案等。
- 邻居反馈排行榜卡片：已展示 `score` 和 `score_reason`。
- `SkillEditor.vue`：Skill 整理编辑器。

**缺失部分**：结算后的奖励分配明细未展示。

## 三、UI 设计方案

### 3.1 展示位置

在悬赏详情区域（或方案列表弹窗下方）新增一个 **“结算报告”卡片**，仅在 `bounty.status === 'completed'` 时显示。

**布局建议**：
```
┌─────────────────────────────────────────────────┐
│  📊 悬赏结算报告                                    │
├─────────────────────────────────────────────────┤
│  总奖励池：10,000 Token    │  参与人数：3          │
├─────────────────────────────────────────────────┤
│  排名   │  节点   │  评分   │  等级  │  获得奖励    │
│  🥇 1   │ node_b │  92分  │   S   │  5,000 Token │
│  🥈 2   │ node_c │  78分  │   B   │  3,000 Token │
│  🥉 3   │ node_d │  65分  │   B   │  2,000 Token │
├─────────────────────────────────────────────────┤
│  评分理由（点击节点查看）                           │
└─────────────────────────────────────────────────┘
```

### 3.2 交互设计
- 结算报告在任务关闭后自动出现在悬赏详情下方。
- 可折叠/展开，避免占用过多空间。
- 点击某个节点可展开详细的评分理由。
- 与现有的“邻居反馈排行榜”区分：排行榜展示所有提交者的评分排序，结算报告额外显示**奖励金额和等级**。

## 四、技术实现步骤

### 4.1 数据获取

**方案 A**：复用 `/close` 接口的返回值。

当前 `closeBounty` 函数中：
```javascript
const res = await request.post(`/bounties/${bounty.id}/close`, {
  issuer_id: props.conversationId
})
// res.data 包含完整的结算结果
```

将 `res.data` 存入一个响应式变量 `settlementResult`，用于渲染结算报告。

**方案 B**：新增独立接口 `GET /bounties/{bounty_id}/settlement`，在查看已完成悬赏时调用。  
（推荐方案 A，因为关闭操作本身就返回了数据，无需额外请求。）

### 4.2 状态管理

在 `BountyMarket.vue` 中添加：
```javascript
const settlementReport = ref(null)  // 存储最近一次结算结果
const showSettlementDialog = ref(false)  // 是否显示结算弹窗/卡片
```

在 `closeBounty` 成功后：
```javascript
settlementReport.value = res.data
showSettlementDialog.value = true  // 或直接渲染在页面上
await fetchBounties()  // 刷新列表状态
```

### 4.3 组件实现

#### 4.3.1 结算报告卡片（内嵌式）

在模板中，于 `neighbor-list` 和悬赏表格之间（或悬赏表格下方）添加：

```vue
<el-card v-if="settlementReport" class="settlement-report" style="margin-bottom: 20px;">
  <template #header>
    <div class="card-header">
      <span>📊 悬赏结算报告</span>
      <el-button type="text" @click="settlementReport = null">关闭</el-button>
    </div>
  </template>

  <div class="report-summary">
    <el-tag type="warning" size="large">总奖励池：{{ settlementReport.reward_pool || '—' }} Token</el-tag>
    <el-tag type="info" size="large">参与人数：{{ settlementReport.submissions_count }}</el-tag>
  </div>

  <el-table :data="settlementReport.evaluation_results" stripe style="margin-top: 16px;">
    <el-table-column type="index" label="排名" width="80">
      <template #default="{ $index }">
        <el-tag :type="$index === 0 ? 'warning' : ''">{{ $index + 1 }}</el-tag>
      </template>
    </el-table-column>
    <el-table-column prop="agent_id" label="节点" width="150" />
    <el-table-column prop="score" label="评分" width="100">
      <template #default="{ row }">
        <el-tag :type="scoreTagType(row.score)">{{ row.score.toFixed(1) }}分</el-tag>
      </template>
    </el-table-column>
    <el-table-column prop="final_grade" label="等级" width="80">
      <template #default="{ row }">
        <el-tag :type="gradeTagType(row.final_grade)" effect="dark">{{ row.final_grade || '—' }}</el-tag>
      </template>
    </el-table-column>
    <el-table-column prop="reward_amount" label="获得奖励" width="120">
      <template #default="{ row }">
        <span class="reward-amount">{{ row.reward_amount || 0 }} Token</span>
      </template>
    </el-table-column>
    <el-table-column prop="reason" label="评分理由" min-width="200" show-overflow-tooltip />
  </el-table>

  <el-divider />
  <div class="curation-info" v-if="settlementReport.curation_results">
    <el-tag type="success">Skill 已沉淀：{{ settlementReport.curation_results.doc_id }}</el-tag>
  </div>
</el-card>
```

#### 4.3.2 辅助函数

```javascript
function gradeTagType(grade) {
  const map = { 'S': 'danger', 'A': 'warning', 'B': 'success', 'C': 'info', 'D': '' }
  return map[grade] || 'info'
}
```

#### 4.3.3 关闭悬赏后自动展示

```javascript
async function closeBounty(bounty) {
  // ... 确认框 ...
  submitting.value = true
  try {
    const res = await request.post(`/bounties/${bounty.id}/close`, {
      issuer_id: props.conversationId
    })
    // 存储结算结果
    settlementReport.value = {
      ...res.data,
      reward_pool: bounty.reward_pool,
    }
    ElMessage.success('任务已结束，奖励已发放！')
    await fetchBounties()
  } catch (err) {
    // ...
  } finally {
    submitting.value = false
  }
}
```

### 4.4 查看历史悬赏的结算报告

对于已经关闭的悬赏（非本次关闭），可通过点击“查看”按钮时，额外请求结算数据。

**后端需补充接口**（简单实现）：
```python
@app.get("/bounties/{bounty_id}/settlement")
async def get_settlement(bounty_id: str):
    # 从 submissions 表中读取 score, final_grade, reward_amount, score_reason
    # 返回格式与 close 接口的 evaluation_results 一致
```

前端调用：
```javascript
async function viewSettlement(bountyId) {
  const res = await request.get(`/bounties/${bountyId}/settlement`)
  settlementReport.value = res.data
}
```

若不想增加后端接口，可复用 `get_submissions` 接口，前端自行聚合数据（但缺少 `final_grade` 字段，可计算或由后端补充）。

## 五、与现有排行榜的整合

现有的“邻居反馈排行榜”在查看方案时展示，侧重于**评分排序**。结算报告侧重于**奖励分配**。

两者可共存：
- 查看方案弹窗中的排行榜保持不变（仅显示 `score` 和 `reason`）。
- 结算报告卡片在任务关闭后显示，包含奖励信息。

## 七、测试验证要点

1. 发布悬赏，邻居提交。
2. 发布者点击“结束任务并结算”。
3. 页面出现结算报告卡片，显示各节点评分、等级、奖励金额。
4. 奖励金额与后端日志一致。
5. 刷新页面后，结算报告消失（不持久化），可再次通过查看历史悬赏时调取。
