# 多Agent多轮研讨闭环技术方案

## 一、方案概述

在现有Nanobot悬赏协作系统之上，将"单次任务执行"升级为"多轮主题研讨"。核心思想是：**将每一轮研讨视为一个悬赏任务**，通过系统自动创建下一轮任务，形成迭代闭环。改造工作主要集中在`BountyHub`和`agent_server.py`，无需新增核心模块。

**核心流程**：
```
初始主题（手动发布）→ 第1轮研讨（悬赏任务）→ 自动评审与聚合 → 生成下一轮议题 → 第2轮研讨 → ... → 收敛或达到max_rounds → 沉淀综合性Skill
```

## 二、架构与数据流

```
┌──────────────────────────────────────────────────────────────────────┐
│                          用户/前端                                     │
│                    发布初始主题（含轮次标记）                            │
└──────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌──────────────────────────────────────────────────────────────────────┐
│                         BountyHub                                     │
│  1. create_bounty (round=1, parent_bounty_id=null)                    │
│  2. close_bounty 时：                                                  │
│     - 评审并聚合观点（调用容器 /aggregate_discussion）                  │
│     - 生成下一轮议题（从聚合结果中提取 next_topic）                     │
│     - 若未收敛且轮次未达上限，自动调用 create_bounty (round+1)        │
│  3. 最终轮结束时，沉淀综合性Skill到 PublicSpace                       │
│  4. 按累计贡献分配奖励池（仅首轮锁仓，后续共享）                        │
└──────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌──────────────────────────────────────────────────────────────────────┐
│                      Agent容器 (发布者)                                │
│  新增端点 /aggregate_discussion：                                      │
│  - 输入：所有提交内容 + 当前轮次 + 历史摘要                            │
│  - 输出：共识摘要、争议焦点、下一轮议题、是否收敛                       │
└──────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌──────────────────────────────────────────────────────────────────────┐
│                        参与Agent容器                                   │
│  无改动，复用现有 /chat 接口处理悬赏任务，利用Session保留上下文        │
└──────────────────────────────────────────────────────────────────────┘
```

## 三、关键决策

### 3.1 max_rounds 默认值
**20轮**

### 3.2 奖励池机制

| 轮次 | 行为 |
|------|------|
| 第1轮 | 锁定全额奖励池 |
| 第2-N轮 | reward_pool=0，不锁仓 |
| 最终轮 | 按累计贡献一次性分配 |

**奖励分配公式**：
```
Agent 最终得分 = Σ (第i轮评分 × 衰减系数^(总轮数-i))
```
- 衰减系数建议：0.9（越早轮次权重略低，鼓励持续深入参与）

### 3.3 收敛判断

```
if converged == true:
    提前终止（优先）
elif current_round < max_rounds:
    继续下一轮
else:
    终止（兜底）
```

**容器端收敛判断标准**：
> "若本轮各 Agent 观点高度一致，未出现新的实质性分歧或补充信息，则设置 `converged=true`。"

### 3.4 上下文传递

每轮描述中注入上一轮共识摘要：
```
[上轮共识]
{consensus}

[待深入]
{next_topic}
```

## 五、关键代码改动

### 5.1 修改 `BountyHub.create_bounty`

增加可选参数 `round_num`、`parent_bounty_id`、`max_rounds`，并支持跳过锁仓逻辑。

```python
async def create_bounty(self, issuer_id: str, title: str, description: str, reward_pool: int, deadline: datetime,
                        docker_reward: int = 0, round_num: int = 1, parent_bounty_id: str = None, max_rounds: int = 20) -> str:
    bounty_id = str(uuid.uuid4())

    # 第1轮悬赏才锁定奖励池，后续轮次共享奖励池
    should_lock_reward = (round_num == 1)

    with get_db() as conn:
        conn.execute("""
            INSERT INTO bounties (id, issuer_id, title, description, reward_pool, docker_reward, deadline, status, created_at, round, parent_bounty_id, max_rounds)
            VALUES (?, ?, ?, ?, ?, ?, ?, 'open', ?, ?, ?, ?)
        """, (bounty_id, issuer_id, title, description, reward_pool, docker_reward, deadline, datetime.now(), round_num, parent_bounty_id, max_rounds))

    # 若需要锁定奖励池
    if should_lock_reward and reward_pool > 0:
        try:
            await self.wallet.transfer(issuer_id, "system", reward_pool, "bounty_lock", bounty_id)
        except Exception as e:
            print(f"[BountyHub] 奖励池锁定失败: {e}")

    return bounty_id
```

### 5.2 修改 `BountyHub.close_bounty`（核心改造）

在任务关闭后判断是否需要启动下一轮研讨或分配累计奖励。

```python
async def close_bounty(self, bounty_id: str, issuer_id: str) -> dict:
    # ... 原有 Step 1~8 不变 ...

    # ========== Step 9：多轮研讨续接 ==========
    bounty = await self.get_bounty(bounty_id)
    current_round = bounty.get("round", 1)
    max_rounds = bounty.get("max_rounds", 20)
    parent_bounty_id = bounty.get("parent_bounty_id")

    # 聚合本轮讨论，获取收敛状态和下一轮议题
    aggregation = await self._aggregate_discussion_and_get_next_topic(bounty_id, issuer_id)
    converged = aggregation.get("converged", False) if aggregation else False

    # 判断是否继续下一轮
    should_continue = False
    if converged:
        print(f"[BountyHub] 讨论已收敛，不再创建下一轮")
    elif current_round < max_rounds:
        consensus = aggregation.get("consensus", "") if aggregation else ""
        next_topic = aggregation.get("next_topic", "") if aggregation else ""
        if next_topic and next_topic != "CONVERGED":
            # 注入上一轮共识到下一轮描述
            enhanced_description = f"[上轮共识]\n{consensus}\n\n[待深入讨论]\n{next_topic}"
            next_bounty_id = await self.create_bounty(
                issuer_id=issuer_id,
                title=f"{bounty['title']} (第{current_round+1}轮)",
                description=enhanced_description,
                reward_pool=0,  # 后续轮次不锁仓
                deadline=datetime.now() + timedelta(hours=24),
                round_num=current_round + 1,
                parent_bounty_id=parent_bounty_id or bounty_id,
                max_rounds=max_rounds
            )
            result["next_bounty_id"] = next_bounty_id
            should_continue = True
            print(f"[BountyHub] 已创建下一轮研讨任务: {next_bounty_id}")

    # ========== Step 10：最终轮奖励分配 ==========
    if not should_continue:
        # 获取根悬赏ID（首轮ID）
        root_bounty_id = parent_bounty_id or bounty_id
        await self._distribute_accumulated_rewards(root_bounty_id, issuer_id)
        # 沉淀综合Skill
        await self._curate_final_discussion_skill(bounty_id, issuer_id)

    return result
```

### 5.3 新增 `_aggregate_discussion_and_get_next_topic`

调用发布者容器的 `/aggregate_discussion` 端点，从所有提交中提炼下一轮议题。

```python
async def _aggregate_discussion_and_get_next_topic(self, bounty_id: str, issuer_id: str) -> dict:
    container_url = self.get_container_url(issuer_id)
    if not container_url:
        return None

    bounty = await self.get_bounty(bounty_id)
    submissions = await self.get_submissions(bounty_id)
    contents = [s["content"] for s in submissions if s.get("content")]

    payload = {
        "bounty_id": bounty_id,
        "bounty_description": bounty.get("description", ""),
        "submissions": contents,
        "round": bounty.get("round", 1)
    }

    try:
        async with aiohttp.ClientSession(timeout=aiohttp.ClientTimeout(total=120)) as session:
            async with session.post(f"{container_url}/aggregate_discussion", json=payload) as resp:
                if resp.status == 200:
                    data = await resp.json()
                    return data
    except Exception as e:
        print(f"[BountyHub] 聚合失败: {e}")
    return None
```

### 5.4 新增 `_distribute_accumulated_rewards`

追溯所有轮次的提交评分，按衰减公式一次性分配奖励。

```python
async def _distribute_accumulated_rewards(self, root_bounty_id: str, issuer_id: str):
    """
    最终轮按累计贡献分配奖励
    公式：Agent 最终得分 = Σ (第i轮评分 × 衰减系数^(总轮数-i))
    """
    DECAY_FACTOR = 0.9

    all_submissions = []
    max_round = 0

    # 追溯所有轮次
    bounty_id = root_bounty_id
    while bounty_id:
        bounty = await self.get_bounty(bounty_id)
        if not bounty:
            break
        round_num = bounty.get("round", 1)
        max_round = max(max_round, round_num)

        submissions = await self.get_submissions(bounty_id)
        for s in submissions:
            s["round"] = round_num
            all_submissions.append(s)

        bounty_id = bounty.get("parent_bounty_id")

    if not all_submissions:
        return

    # 按agent_id聚合评分
    agent_scores = {}
    for s in all_submissions:
        agent_id = s.get("agent_id")
        score = s.get("score", 0) or 0
        if agent_id not in agent_scores:
            agent_scores[agent_id] = []
        agent_scores[agent_id].append((s["round"], score))

    # 计算加权得分
    weighted_scores = {}
    for agent_id, scores in agent_scores.items():
        total_weighted = 0
        for round_num, score in scores:
            weight = DECAY_FACTOR ** (max_round - round_num)
            total_weighted += score * weight
        weighted_scores[agent_id] = total_weighted

    # 获取奖励池金额
    with get_db() as conn:
        row = conn.execute("SELECT reward_pool FROM bounties WHERE id = ?", (root_bounty_id,)).fetchone()
        reward_pool = row[0] if row else 0

    # 按比例分配
    total_score = sum(weighted_scores.values())
    if total_score == 0:
        return

    reward_results = []
    for agent_id, weighted_score in weighted_scores.items():
        ratio = weighted_score / total_score
        amount = int(reward_pool * ratio)
        try:
            await self.wallet.transfer("system", agent_id, amount, "bounty_reward", root_bounty_id)
            reward_results.append({"agent_id": agent_id, "reward_amount": amount, "status": "success"})
        except Exception as e:
            reward_results.append({"agent_id": agent_id, "reward_amount": 0, "status": "failed", "error": str(e)})

    print(f"[BountyHub] 累计奖励分配完成: {reward_results}")
    return reward_results
```

### 5.6 最终沉淀综合性Skill

在最后一轮结束时调用（可复用现有 `_curate_aggregated_report` 或增强）。

```python
async def _curate_final_discussion_skill(self, final_bounty_id: str, issuer_id: str):
    """沉淀多轮研讨的综合Skill"""
    bounty = await self.get_bounty(final_bounty_id)

    # 收集所有轮次的提交内容
    all_contents = []
    bounty_id = final_bounty_id
    rounds_data = []

    while bounty_id:
        bounty = await self.get_bounty(bounty_id)
        if not bounty:
            break
        submissions = await self.get_submissions(bounty_id)
        for s in submissions:
            if s.get("content"):
                all_contents.append(s["content"])
                rounds_data.append({
                    "round": bounty.get("round", 1),
                    "content": s["content"],
                    "agent_id": s.get("agent_id")
                })
        bounty_id = bounty.get("parent_bounty_id")

    # 调用聚合生成综合Skill
    skill_report = await self._curate_aggregated_report(
        task_description=bounty.get("description", ""),
        submissions=all_contents,
        evaluation_results=[{"score": s.get("score", 0)} for s in rounds_data]
    )

    # 保存Skill
    skill_dir = Path("xuanshang_report") / f"multi_round_skill_{final_bounty_id}"
    skill_dir.mkdir(parents=True, exist_ok=True)
    (skill_dir / "SKILL.md").write_text(skill_report)
    print(f"[BountyHub] 多轮研讨Skill已沉淀: {skill_dir}")
    return str(skill_dir / "SKILL.md")
```

## 六、验证方案

1. **手动发布初始主题**：描述中注明"本任务将进行多轮研讨，请各Agent就XX话题发表观点"。
2. **观察日志**：
   - 第一轮结束后，应自动创建第二轮悬赏，标题后缀"(第2轮)"
   - 描述中应包含"[上轮共识]"段落
3. **收敛测试**：若各Agent观点一致，应在达到max_rounds前提前终止
4. **奖励验证**：检查最终奖励是否按衰减公式分配
5. **Skill沉淀**：检查多轮研讨综合Skill是否正确生成

## 七、注意事项

- Token消耗会增加，max_rounds=20是上限，建议根据实际情况调整
- 各Agent容器的Session能跨轮次保留上下文，通过下一轮描述中的共识摘要实现认知连贯
- 衰减系数0.9可根据讨论质量灵活调整
- 建议在讨论初期设置较高的奖励池，以吸引更多Agent参与
