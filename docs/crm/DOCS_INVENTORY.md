# CRM Documentation Inventory

Audit date: 2026-05-08

Decision context: real CRM access is moving from an in-process Nanobot `RealCRMAdapter` that talks directly to GraphQL to a separate read-only CRM MCP Server. This inventory classifies existing CRM-related documentation and task artifacts so the MCP route can proceed from the canonical `docs/crm/` directory without deleting or moving historical files.

Task 15J phase one added `docs/crm/REAL_ADAPTER_CLEANUP_REVIEW.md` as the cleanup decision document. Task 15K executes approved Option B: archive and mark superseded direct GraphQL docs in place, keep direct adapter code/tests as reference, and do not delete or move files. The direct adapter code and tests are superseded-reference material, not the production CRM access route.

Scope scanned:

- `.dek/project/`
- `.dek/changes/crm-opportunity-intelligence/`
- `.dek/tasks/crm-ai-analysis-layer/`
- `docs/`
- `nanobot/skills/crm-opportunity-intelligence/SKILL.md`

Safety notes:

- No files are approved for deletion in this audit.
- No business code was changed for this audit.
- No MCP server was implemented for this audit.
- `.env.nanobot` was not read.
- No real CRM endpoint was accessed.
- No token, secret, or real CRM data is recorded here.
- `docs/crm/README.md` is now the canonical CRM documentation entry point.
