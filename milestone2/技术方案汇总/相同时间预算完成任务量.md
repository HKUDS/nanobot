好的，以下是一个针对“**相同时间预算下任务完成数量对比**”的实验方案，旨在剥离长尾推理的干扰，直接测量 SAyG-Mem 的吞吐量优势。

---

## 实验名称
**固定时间窗口内的最大推理轮次对比实验**

## 实验目的
在给定相同时间预算（例如 300 秒）的条件下，对比 SAyG-Mem（A 组）与基线（B 组）各能完成多少轮 Agent 推理任务，以量化 CWW 异步合并机制对系统吞吐量的真实提升。

## 实验设计核心思想
- **固定时间，不固定轮次**：传统实验是固定 5 轮测耗时，本实验反过来——固定总耗时预算，统计每个 Agent 实际完成的轮数。
- **消除长尾干扰**：由于每个 Agent 独立连续推理，快者可以完成更多轮，慢者完成较少轮，但总吞吐量由所有 Agent 的累加轮次决定，避免单个慢 Agent 拖垮整体耗时指标。
- **直观体现 CWW 优势**：A 组无同步栅栏，Agent 可全速连续推理，理论累加轮次应显著高于 B 组（B 组每轮末有强制同步等待）。

## 实验设置

### 公共参数
| 参数 | 值 |
|------|-----|
| 时间预算 `TIME_BUDGET` | 300 秒（5 分钟） |
| Agent 数量 | 10 个 |
| Agent 角色 | 沿用原 10 种角色，确保任务多样性 |
| 推理模型 | `deepseek-chat` |
| 预热 | 预置 0 号 Skill，与之前相同 |

### A 组（SAyG-Mem）配置
- 每个 Agent 独立、连续执行推理任务，**无轮次上限**，直到总实验时间达到 `TIME_BUDGET` 为止。
- 合并触发：KM 后台自动监控全局未合并堆段数，达到阈值（默认 20）或定时触发合并，完全异步，不阻塞 Agent。
- 实验结束条件：从第一个 Agent 开始推理起计时，到达 `TIME_BUDGET` 后立即终止所有 Agent 容器。

### B 组（Baseline）配置
- 同样 10 个 Agent 并发推理，但采用**轮次栅栏**模式：每一轮所有 Agent 完成推理后，执行一次同步 LLM 融合（合并），再集体进入下一轮。
- 实验结束条件：到达 `TIME_BUDGET` 后，完成当前正在进行的轮次（即等待该轮所有 Agent 完成并执行合并后）停止，统计完成的完整轮数及额外完成的零星轮次（若有）。

## 度量指标
| 指标 | 定义 | 预期 A 组优势 |
|------|------|-------------|
| **总完成任务轮数** | 所有 Agent 完成的推理轮次之和 | 显著更高（因无同步等待） |
| **平均单 Agent 完成轮数** | 总轮数 / Agent 数量 | 更高 |
| **合并/融合总耗时占比** | 合并耗时 / 总时间预算 | 极低（A 组仅 1~2 次合并，B 组每轮都有） |
| **Agent 空闲等待占比** | （总时间 - 累加推理时间）/ 总时间 | A 组远低于 B 组 |
| **PublicMemory 最终条目数** | 实验结束时知识库条目数 | 反映知识积累效率 |

## 脚本修改要点

### 1. 新增时间预算控制
在原 `single_agent_learning` 函数外层增加全局截止时间判断：
```python
EXPERIMENT_DEADLINE = time.perf_counter() + TIME_BUDGET

async def single_agent_learning_unbounded(agent_info, role):
    round_idx = 0
    while time.perf_counter() < EXPERIMENT_DEADLINE:
        round_idx += 1
        # 执行一轮推理（复用原逻辑）
        success = await do_one_round(agent_info, round_idx, role)
        if not success:
            # 可记录失败但继续尝试，直到时间耗尽
            pass
    return round_idx
```

### 2. B 组轮次栅栏改造
B 组需要支持无限轮次，直至时间耗尽：
```python
round_idx = 0
while time.perf_counter() < EXPERIMENT_DEADLINE:
    round_idx += 1
    # 并发执行本轮所有 Agent 推理
    round_tasks = [do_one_round(agent, round_idx, role) for agent in agents]
    await asyncio.gather(*round_tasks)
    # 检查是否超时（若超时则不计入完整轮次，但可统计已完成的）
    if time.perf_counter() >= EXPERIMENT_DEADLINE:
        break
    # 执行同步合并
    await llm_merge_current_round()
```

### 3. 实验结果记录
- 记录每个 Agent 实际完成的轮数（包括未完整轮次的部分计数策略，需明确定义）。
- 记录总耗时（应略超 `TIME_BUDGET`，因需完成当前原子操作）。
- 记录合并触发次数及总耗时。
- 导出 PublicMemory 并统计条目数。

## 预期结果与解读

假设单轮推理时间平均 30 秒，B 组每轮合并耗时 7 秒，则：
- B 组每轮总耗时 ≈ 37 秒，300 秒内约完成 **8 轮**（10 Agent × 8 = 80 轮次）。
- A 组 Agent 无合并等待，300 秒内每个 Agent 约完成 10 轮（300 / 30），累加约 **100 轮次**，提升 **25%**。

若进一步降低合并阈值或优化合并开销，优势将更明显。
